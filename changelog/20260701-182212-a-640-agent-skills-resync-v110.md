## Changed

- Re-vendored all seven installed agent-skills bundles (`send-it`, `preflight`,
  `changelog`, `linear-sync`, `cleanup-repo`, `triage-pr`, `initialise-skills`) to
  agent-skills v1.1.0, mirrored across `.claude/skills/` and `.agents/skills/`, and
  refreshed `skills-lock.json`. Bundle versions: `send-it` 0.5.0 → 0.5.1,
  `changelog` 0.8.0 → 0.8.1, `cleanup-repo`/`linear-sync` → 0.3.3,
  `triage-pr` → 0.5.1 (`preflight` 0.2.0 and `initialise-skills` 0.6.1 were already
  current).
- Adopted the **generated-config model** (agent-skills `c2a7fdb`): per-skill
  `config.json` is no longer committed. The previously-tracked configs were removed
  and `.claude/skills/*/config.json` + `.agents/skills/*/config.json` are now
  gitignored, so the template no longer carries its own values into spawned repos —
  the neutral `config.example.json` is the only config that ships. Ran the
  `initialise-skills` skill to regenerate this repo's local (untracked) configs from
  detected facts (`dist` shippable surface, `A` issue-key prefix, `ACME Skunkworks`
  team, `acme-skunkworks` workspace); a second dry-run is a stable no-op.
- Behavioural changes riding along in v1.1.0 (documented in the docs refresh):
  `send-it` decides release-type by semantic category (the `changelogScope` knob is
  gone), `preflight` gains `blockOnWarnings` (errors-only by default), and
  `changelog` add-links is branch-scoped by default.
